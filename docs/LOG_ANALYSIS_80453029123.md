# Análise do workflow 80453029123

## Resultado geral

Os jobs PHP 8.3, PHP 8.4, MySQL 8.4, PostgreSQL 17, contratos Docker/Compose e CloudPanel nativo foram concluídos. A falha ficou isolada no container FreeRADIUS do Docker Playground.

## Causa-raiz

O módulo SQL passou a ser localizado corretamente em:

```text
/etc/freeradius/mods-enabled/sql
```

Entretanto, a linha 14 do template continha múltiplas atribuições sem separação por nova linha:

```text
pool { start = 5 min = 3 max = 20 spare = 5 uses = 0 retry_delay = 30 lifetime = 0 idle_timeout = 60 }
```

O parser 3.2.10 interrompeu a inicialização com:

```text
Syntax error: Expected comma after '5'
```

## Correção 1.4.2

- transforma o bloco `pool` em estrutura multilinha;
- adiciona `max_retries` e `cleanup_interval`;
- valida os templates com um verificador PHP;
- executa o parser real `freeradius -XC` durante o build da imagem;
- valida PostgreSQL e MySQL separadamente;
- mantém a validação integral obrigatória da PR;
- exibe a configuração SQL com senha mascarada quando o parser falhar;
- incrementa a versão para 1.4.2.
