# Análise do workflow 80457568511

## Resultado geral

Foram aprovados:

- PHP 8.3;
- PHP 8.4;
- MySQL 8.4;
- PostgreSQL 17;
- contratos Docker e Compose;
- CloudPanel nativo.

A única falha ocorreu no build da imagem FreeRADIUS, antes da inicialização do Playground.

## Causa-raiz

O parser real foi executado durante o build e carregou corretamente:

```text
mods-enabled/sql
mods-config/sql/main/postgresql/queries.conf
sites-enabled/default
```

Em seguida, recusou o virtual server:

```text
sites-enabled/default[29]: Parse error after "pap": unexpected token "}"
Errors reading or parsing radiusd.conf
Template FreeRADIUS inválido para postgresql.
```

O bloco `authenticate` utilizava `Auth-Type PAP { pap }`, `Auth-Type CHAP { chap }` e `Auth-Type MS-CHAP { mschap }` em linha única. A sintaxe compactada era aceita pelo verificador estático anterior, mas não pelo parser FreeRADIUS 3.2.10 usado na imagem.

## Correção 1.4.3

- converte os três blocos `Auth-Type` para formato multilinha;
- valida chaves balanceadas do virtual server;
- rejeita blocos nomeados compactados;
- exige PAP, CHAP e MS-CHAP com seus módulos correspondentes;
- adiciona teste unitário específico do virtual server;
- mantém o parser real para PostgreSQL e MySQL no build;
- mantém Playground, login, Access-Accept, Accounting-Response e CloudPanel obrigatórios na PR.
