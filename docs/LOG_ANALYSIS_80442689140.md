# Análise do workflow 80442689140

## Resultado geral

Os jobs MySQL 8.4, PostgreSQL 17 e as imagens Docker `app`, `web` e `freeradius` concluíram sem erro funcional. As falhas ficaram concentradas em quatro jobs e foram agrupadas em três causas-raiz.

## 1. PHP 8.3 e PHP 8.4 — PlaygroundSeederTest

Erro observado:

```text
No query results for model [App\Models\User].
tests/Feature/PlaygroundSeederTest.php:46
```

A linha procurava sempre `admin@playground.local`, enquanto o seeder obtinha o e-mail de `config('playground.seed.admin_email')`. No ambiente de teste, essa configuração podia herdar `SEED_ADMIN_EMAIL` do `.env.testing`, gerando outro e-mail.

Correção:

- o teste define explicitamente os três e-mails de demonstração;
- a asserção usa a própria configuração do playground;
- o comportamento produtivo do seeder não foi alterado.

## 2. Docker Playground — container app encerrado

Erro observado:

```json
{
  "ok": false,
  "error": "Simulador MikroTik indisponível no container de serviços."
}
```

O construtor de `MikrotikSshService` mantinha o simulador como dependência opcional. A resolução automática utilizava o valor padrão `null`, e o comando `radiushub:playground:verify` retornava código de falha durante o entrypoint.

Correção:

- foi adicionado binding explícito de `MikrotikSshService` no `AppServiceProvider`;
- o binding injeta `MikrotikSimulatorService` no serviço gerenciado pelo Laravel;
- o parâmetro opcional foi preservado para não quebrar testes ou integrações que instanciam o serviço manualmente;
- foi adicionado teste de regressão do binding.

## 3. CloudPanel nativo — cache antes das migrations

Erro observado:

```text
SQLSTATE[42P01]: Undefined table: relation "cache" does not exist
SQL: delete from "cache"
```

O instalador executava `php artisan optimize:clear` com `CACHE_STORE=database` antes de criar a tabela `cache`.

Correção:

- foi criada `artisan_optimize_clear_safe` em `scripts/lib.sh`;
- durante a limpeza inicial, são usados temporariamente:
  - `CACHE_STORE=array`;
  - `CACHE_LIMITER=array`;
  - `SESSION_DRIVER=array`;
  - `QUEUE_CONNECTION=sync`;
- as configurações reais do `.env` não são alteradas;
- migrations continuam sendo executadas na mesma ordem e com o banco configurado;
- instaladores e atualizadores atuais reutilizam a mesma rotina segura.

## Compatibilidade

Não foram alteradas migrations, tabelas, rotas públicas, contratos de API, regras de autenticação, integração Asaas, FreeRADIUS ou comunicação SSH real com MikroTik.
